## 1. Цель проекта
Целью проекта было разработать архитектуру системы с использованием **абстрактных базовых классов (АБК)** для определения контрактов поведения (интерфейсов), обеспечивая полиморфную реализацию обязательных методов различными классами.

## 2. Описание интерфейсов
- **`ITrainable`**: Требует метод `execute_workout()`. Определяет контракт для любого объекта, который может выполнять обучение.

- **`IReportable`**: Требует метод `get_summary()`. Определяет контракт для объектов, которые генерируют информационные данные.

## 3. Внедрение в учебные группы
- **Профессионал-спортсмен**: Реализованы оба интерфейса. Тренировка длительная, в отчете освещаются вопросы спонсорства.
