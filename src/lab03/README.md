## Цель практики
В центре внимания этой лабораторной работы — эволюция сущности «Спортсмен» посредством наследования, что позволяет специализировать поведение и применять чистый полиморфизм для избежания сложных условных структур.

## Прикладные концепции:
Наследование: Создание специализированных подклассов из базового класса.
Повторное использование кода: Использование `super()` для вызова конструктора и методов базового класса.
Полиморфизм: Реализация методов с одинаковой сигнатурой, но разным поведением.
Принцип DRY (Don't Repeat Yourself — не повторяйте себя): Интеграция лабораторных работ 01 и 02 без дублирования файлов.

## Структура класса
  1. Базовый класс: Спортсмен (из LP-1)
Предоставляет основные характеристики: имя, возраст, вид спорта, вес, рост и рейтинг.

2. Подкласс: Профессиональный спортсмен
Представляет спортсменов высокого уровня.

Новые атрибуты: зарплата и спонсор.

Новый метод: promote_sponsor() для маркетинга.

Полиморфизм: расчет производительности с учетом рейтингового бонуса в зависимости от профессионального уровня.
3. Подкласс: AmateurAthlete

Представляет спортсменов-любителей или профессиональных спортсменов.
Новые атрибуты: club (местный клуб) и occupation (профессия/учеба).
Новый метод: get_club_info() для получения подробной информации о клубе.
Полиморфизм: упрощенный расчет производительности на основе базового рейтинга.

## Технические особенности (Уровень 5)

Полиморфизм без условных операторов: метод calculate_performance() вызывается в цикле for для AthleteCollection. Python автоматически выполняет правильную логику без необходимости использования конструкции if type == ....
Интеграция с лабораторной работой 02: коллекция AthleteCollection хранит и управляет объектами разных типов из одной и той же иерархии прозрачно.
Единый путь импорта: надежная конфигурация через sys.path в файле base.py для предотвращения конфликтов идентичности классов.
